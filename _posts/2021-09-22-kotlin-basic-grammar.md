## <span style="color:#0f7b6c">1. 코틀린 언어의 개요</span> 
- Intellj IDEA를 제작한 JetBrains에서 만든 프로그래밍 언어
- 2011년에 공개된 이후, 2017년부터 안드로이드 개발 공식 언어로 구글이 코틀린을 소개
- 현재 Java 대체 언어로 많은 주목을 받는 중
- [https://kotlinlang.org/](https://kotlinlang.org/)

#### 1-1. 코틀린 언어의 동작 원리
![kotlin principle of operation]({{site.url}}/img/Kotlin/kotlin-principle-operation.png){: height="300"} <br>
> 코틀린은 100% 개발자를 위한 코드다.

자바의 경우 JVM을 위한 코드이고, 바이너리 코드의 경우 컴퓨터를 위한 코드라면 코틀린은 개발자를 위해 고안된 코드이다. <br>
때문에 코틀린 언어로 만들어진 프로그램의 경우 개발자의 필요에 따라 자바, 자바스크립트, C 코드로 변경될 수 있다. <br>
코틀린 언어의 최종 목표는 **'모든 프로그래밍 분야에서 코틀린 언어를 사용할 수 있도록 만드는 것'** 이다.

#### 1-2. 코틀린 언어의 특징
- 최선 프로그래밍 언어 답게 매우 간단하고 편리한 문법을 제공
- Null 안전성을 위해 Null 허용 변수와 그렇지 않는 변수를 구분
- 예외 처리를 강제하지 않음
- 모든 자료형 값을 객체로 관리
- OOP와 Functional Programming 모두 지원

## <span style="color:#0f7b6c">2. 변수와 자료형</span>
>**변수**란 프로그래밍 과정에서 사용할 값을 저장하기 위한 공간이다. <br><br>
**자료형**이란 값을 저장 공간에 어떤 방식으로 저장할지 정의하는 것 이다.

프로그래밍 과정에서 값을 사용하는 것은 당연하기 때문에 변수의 존재는 이해가 간다. 하지만 자료형은 왜 필요한지 이해가 잘 안갈 수 있다. 왜 필요할까?? <br><br>
🔔컴퓨터가 생각보다 똑똑하지 않기 때문이다. 컴퓨터의 본질은 0과 1이 끝이다. 그 밖에는 아무것도 모른다. 내가 숫자 100을 저장해도 컴퓨터 내부에서는 사실 0과 1이고, 내가 "조윤환"이라는 값을 저장해도 컴퓨터 내부에서는 역시나 0과 1이다. 그렇기 때문에 자료형이 필요한 것이다. 똑같이 0과 1로 저장된 값이라도 자료형에 따라 프로그래머는 목적에 맞게 사용할 수 있다.

#### 2-1. 리터럴
> 코드 작성시 값을 작성하는 문법

|리터럴|내용|설명|
|---|---|---|
|**정수**|100 , 100L , 1_000_000|21억이 넘어가는 숫자도 L이 필수가 아니다|
|**실수**|11.11 , 11.11F|아무것도 붙이지 않았을 때는 DOUBLE(8byte) 타입이다. |
|**문자**|'a' , 'b'|-|
|**문자열**|"hello world" , """여러줄 문자열"""|-|
|**논리**|true false|-|

#### 2-2. 자료형
> 코틀린은 모든 자료형 값을 객체로 처리한다. 

|자료형|종류|
|---|---|
|**정수**|Long(8) , Int(4) , Short(2) , Byte(1)|
|**부호가 없는 정수**|ULong(8) , UInt(4) , UShort(2) , UByte(1)|
|**실수**|Double(8) , Float(4)|
|**논리**|Boolean(1)|
|**문자**|Char(2)|
|**문자열**|String|

#### 2-3. 변수 선언

- **var(variable)**는 선언 이후 값을 다시 저장할 수 있다.
- **val(value)**는 선언 이후 값을 변경할 수 없다. 
- var/val 변수명 : 타입 = 값
- 타입 생략 시 저장하는 값에 따라 자료형이 자동으로 결정된다.

#### 2-4. Null 허용 변수

- 코틀린은 변수를 선언 할 때 null 허용 여부를 설정할 수 있다.
- var/val 변수명:자료형 &nbsp;= 값 - null을 허용하지 않는 변수
- var/val 변수명:자료형? = 값 - null을 허용하는 변수

<pre><code>
    var a0 : Int = 10
    var a1 : Int? = null 

    var a2 : Int = a0 // null을 허용하지 않는 변수에 null을 허용하지 않는 다른 변수의 값을 넣을 수 있다.
    a2 = a1 // null을 허용하지 않는 변수에 null을 허용하는 다른 변수의 값을 넣을 수 없다.

    var a3 : Int? = a0 // null을 허용하는 변수에 null을 허용하지 않는 다른 변수의 값을 넣을 수 없다.
    a3 = a1 // null을 허용하는 변수에 null을 허용하는 다른 변수의 값을 넣을 수 있다.
</code></pre> 

null을 허용하는 변수가 null값이 아닌 다른 값을 가지고 있을 때, !! 연산자를 이용해 null을 허용하지 않는 변수로 변경할 수 있다.

## <span style="color:#0f7b6c">3. 함수</span>
코틀린 함수의 기본 문법
<pre><code>
    fun 함수명(매개변수) : 반환값 타입 {
        // to do
    }
    
    // 매개변수의 경우 "변수명:자료형" 형태로 정의한다.
    // 반환값 타입을 생략할 경우 void(Unit)이다.
</code></pre>

#### 3-1. 함수의 오버로딩
> 함수의 오버로딩이란 매개변수의 형태를 다르게 하여 같은 이름으로 여러 함수를 정의하는 개념이다. <br>
매개변수의 개수나 타입이 다른 경우에 해당하며, 리턴타입이 다른 경우는 해당하지 않는다.

<pre><code>
    fun test1(){
        
    }

    fun test1(a1:Int){
        
    }

    fun test1(a1:Double){
        
    }
</code></pre>

#### 3-2. 지역 함수
- 함수 내에 정의한 함수
- 함수를 정의한 함수 안에서만 호출이 가능하다.

<pre><code>
    fun testFun(){
        println("test function 호출")

        fun testLocalFun(){
            println("test local function 호춡")
        }

        testLocalFun() // main과 같이 외부에서는 호출할 수 없다.
    }
</code></pre>

## <span style="color:#0f7b6c">4. 제어문</span>
- 프로그램 코드는 위에서 아래로 흘러간다. ↓
- 이 흐름을 변경하고자 할 때 사용하는 것이 제어문이다.
- 제어문은 **분기문**과 **반복문**으로 구분된다.

#### 4-1. if
<pre><code>
    if (조건){
        // 조건을 만족했을 때 실행할 코드
    } else {
        // 조건을 만족하지 않을 때 실행할 코드
    }
</code></pre>

if문을 활용한 변수 초기화
<pre><code>
    var a1: String = if(조건) "조건이 참일 때" else "조건이 거짓일 때"
</code></pre>

#### 4-2. When
- When은 Java의 switch case와 비슷하다.
- 수식이나 값의 결과에 따라 수행될 부분을 결정하게 되며 자바와 다르게 정수 뿐만 아니라 다양한 타입(실수, 정수, 문자열 ..)의 값을 비교할 수 있다.

<pre><code>
    when(변수 or 수식){
        1 -> {
            // to do
        }
        2 -> {

        }
        3 -> {
            
        }
        4, 5 -> {
            // 4또는 5 일 때 
        }
        in 6..10 -> {
            // 6과 10 사이의 값 일 때
        }
        else -> {

        }
    }
</code></pre>

when을 활용한 변수 초기화
<pre><code>
    fun setValue(a1:Int) = when(a1){
        1 -> "문자열1"
        2 -> "문자열2"
        else -> "그 외 문자열"
    }

    var str1 = setValue(1) // str1이 "문자열1"로 초기화 된다.
</code></pre>
